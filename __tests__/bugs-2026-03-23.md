# Bugs — 2026-03-23

> Jarvis Horner — audyt frontendu i backendu DQN Trading Bot
> Kod: `server/index.js`, `client/index.html`, `src/`

## BUG-001: [Frontend] Stan przycisków nie synchronizuje się z serwerem po odświeżeniu
- **Severity:** ważny
- **Lokalizacja:** `client/index.html` ~linia 1796
- **Kroki:**
  1. Kliknij "▶ TRAIN" — trening się uruchamia, przycisk zmienia na "⏹ STOP"
  2. Odśwież stronę (F5)
- **Oczekiwane:** Przycisk pokazuje "⏹ STOP" (serwer wciąż trenuje)
- **Rzeczywiste:** Przycisk pokazuje "▶ TRAIN" — stan `trading` jest zmienną lokalną w `onclick`, nie jest odczytywana z serwera przy ładowaniu strony
- **Kod:** `let trading` nie istnieje globalnie; `document.getElementById('trainBtn').onclick=async()=>{ if(!trading){ ... } }` — `trading` jest tworzony dopiero po pierwszym kliknięciu

## BUG-002: [Frontend] Brak ładowania stanu Paper Trading przy starcie strony
- **Severity:** średni
- **Lokalizacja:** `client/index.html`
- **Kroki:**
  1. Włącz Paper Trading
  2. Odśwież stronę
- **Oczekiwane:** Dashboard pokazuje aktualny stan Paper Trading (pozycje, kapitał)
- **Rzeczywiste:** Dashboard nie wywołuje `/api/paper-trading/status` przy ładowaniu — dane mogą być nieaktualne do pierwszego pollingu

## BUG-003: [Backend] Brak obsługi błędów w `updatePrices()` — ciche porażki
- **Severity:** średni
- **Lokalizacja:** `server/index.js` — `updatePrices()`
- **Kod:**
  ```javascript
  async function updatePrices() {
      try {
          const [btc, eth, sol] = await Promise.all([...]);
          prices = { btc: parseFloat(btc.price), ... };
      } catch (e) {}  // ← pusty catch, zero logowania
  }
  ```
- **Kroki:** Zablokuj dostęp do api.binance.com (firewall/offline)
- **Oczekiwane:** Log ostrzeżenia, ceny oznaczone jako niedostępne
- **Rzeczywiste:** Ciche porażka co 3 sekundy, `prices` zostaje `{btc: 0, eth: 0, sol: 0}` — Paper Trading nie działa ale nie informuje użytkownika

## BUG-004: [Backend] Brak walidacji SL/TP po stronie serwera
- **Severity:** średni
- **Lokalizacja:** `server/index.js` — `/api/paper-trading/sltp`
- **Kod:**
  ```javascript
  if (!isNaN(sl) && sl > 0 && sl <= 50) paperTrading.stopLoss = sl;
  if (!isNaN(tp) && tp > 0 && tp <= 100) paperTrading.takeProfit = tp;
  ```
- **Problem:** Serwer waliduje zakres, ale nie waliduje relacji SL/TP. Można ustawić SL=50% i TP=0.1% (RR 1:500 na niekorzyść). Brak ostrzeżenia przed absurdalnymi wartościami.

## BUG-005: [Frontend] Brak globalnej obsługi błędów fetch — crash na timeout/500
- **Severity:**ważny
- **Lokalizacja:** `client/index.html` — wiele handlerów onclick
- **Kroki:**
  1. Zatrzymaj serwer (port 3000 niedostępny)
  2. Kliknij dowolny przycisk (TRAIN, SAVE, Paper START itd.)
- **Oczekiwane:** Komunikat błędu "Serwer niedostępny", przycisk nie blokuje się
- **Rzeczywiste:** Uncaught Promise rejection w konsoli, przycisk może się zablokować w stanie pośrednim

## BUG-006: [Backend] `trainingStep()` wywoływana co 100ms — potencjalny bottleneck
- **Severity:** kosmetyczny / wydajnościowy
- **Lokalizacja:** `server/index.js` — `setInterval(trainingStep, 100)`
- **Problem:** Każdy krok treningowy (act → execute → remember → replay) jest wykonywany co 100ms. Jeśli jeden krok trwa > 100ms, kroki się nakładają. Brak mutex/flagi `isStepping`.
- **Kod:** `agent.replay()` jest async ale `setInterval` nie czeka na Promise

## BUG-007: [Frontend] Monolityczny plik HTML (4300+ linii) — maintainability
- **Severity:** kosmetyczny / tech debt
- **Lokalizacja:** `client/index.html`
- **Problem:** Cały frontend (HTML + CSS + JS) w jednym pliku. 4300+ linii. Brak modularizacji, brak minifikacji, brak source map. Trudne do testowania i debugowania.

## BUG-008: [Backend] Brak rate limiting na endpointach API
- **Severity:** średni / security
- **Lokalizacja:** `server/index.js`
- **Problem:** Brak rate limiting na żadnym endpointzie. Klient może spamować `/api/training/start`, `/api/paper-trading/toggle` itd. bez ograniczeń.

## BUG-009: [Backend] `prevEpData` aktualizowane tylko gdy >= 3 candle
- **Severity:** niski
- **Lokalizacja:** `server/index.js` — sekcja episode end
- **Kod:**
  ```javascript
  if (currentEpData.length >= 3) {
      prevEpData = [...currentEpData];
  }
  ```
- **Problem:** Jeśli epizod zakończy się bardzo szybko (<3 kroków), `prevEpData` nie zostanie zaktualizowane — wykres "Poprzedni epizod" pokaże dane z jeszcze wcześniejszego epizodu.

## BUG-010: [Backend] Brak autentyfikacji na endpointach kontrolnych
- **Severity:**ważny / security
- **Lokalizacja:** `server/index.js` — WSZYSTKIE endpointy
- **Problem:** Każdy kto zna adres IP:port może wywołać `/api/training/start`, `/api/training/stop`, `/api/paper-trading/reset`, `/api/live-trading/start` itd. Brak auth token, brak CORS restriction (ustawione `*`).

## BUG-011: [Frontend] Scroll do góry po każdym odświeżeniu — brak pozycji
- **Severity:** kosmetyczny
- **Lokalizacja:** `client/index.html`
- **Kroki:**
  1. Przewiń stronę w dół
  2. Poczekaj na automatyczne odświeżenie danych
- **Oczekiwane:** Strona pozostaje na tej samej pozycji
- **Rzeczywiste:** Strona przeskakuje do góry (DOM update powoduje scroll reset)

## BUG-012: [Backend] Model nie jest zapisywany przy graceful shutdown (SIGTERM)
- **Severity:** średni
- **Lokalizacja:** `server/index.js` — `process.on('SIGTERM')`
- **Kod:**
  ```javascript
  process.on('SIGTERM', () => { server.close(); process.exit(0); });
  ```
- **Problem:** Przy SIGTERM serwer zamyka się natychmiast bez zapisania modelu. Postęp treningu jest tracony.
